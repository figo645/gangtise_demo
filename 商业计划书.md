## 1. 执行摘要

洞见智研是一款聚焦证券研究场景的智能投研产品，核心不是做“泛聊天 AI”，也不是做“泛金融 SaaS”，而是围绕高频、刚需、可复用的研究动作，构建一套能够真正服务普通投资者、大V 投顾租户和平台管理方的智能工作流系统。

当前产品已形成三端清晰分工：

- H5 用户端：承接普通投资者的基本面分析、自选股、复盘内容消费、Hermes 对话与消息触达。
- 大V 工作台 Web：承接大V 的粉丝经营、复盘生产、知识库微调、Skills 绑定、Hermes 使用、核心指标配置与经营 Dashboard。
- 纯 Admin 后台：承接租户管理、能力控开、审计、一致性巡检、平台级分析与经营配置。

系统当前最核心的差异化能力，不是单点分析结果，而是“知识库 + Skills + Hermes + 复盘/分析输出”的闭环：

- 知识库负责沉淀租户专属研究资料与历史方法论。
- Skills 负责将知识库、提示词、结构化输出方式组织成可复用的研究能力单元。
- Hermes 负责以对话式或工作区式交互，调用当前 Skill 与知识上下文输出图表加内容的结构化回答。
- 大V 工作台负责把这些能力用于复盘、粉丝经营、内容生产与商业转化。

从商业上看，洞见智研具备三条清晰收入主线：

- C 端订阅与算力包收入。
- 大V 工作台与租户经营服务收入。
- 定制化研究工具、机构试点与后续 API/知识部署收入。

本项目当前阶段的目标，不是追求“大而全”，而是用最少但最有价值的场景建立高留存、高复用、高复盘效率的产品基础，再逐步放大收入与租户规模。

当前的起盘路径也已经明确：先以大V 作为切入点，用大V 的复盘、答疑、Hermes 使用与知识经营能力，承接其原有粉丝客群和新增粉丝客群，再将这些 C 端用户沉淀为可长期经营的研究型社区资产。

## 2. 我们要解决什么问题

### 2.1 普通投资者的问题

普通投资者在证券研究场景里常见的痛点有：

- 信息分散：研报、纪要、新闻、社媒观点分散在多个渠道，难以统一整理。
- 表达过载：大量研究内容篇幅长、结构乱、无法快速抓住关键判断。
- 跟踪成本高：自选股很多，但缺少统一的跟踪逻辑、更新机制和复盘入口。
- 对 AI 不信任：通用聊天工具往往“说得很多”，但来源、证据链与判断边界不清晰。
- 缺少持续性：即使看完内容，也很难把每日研究动作沉淀成自己的长期方法。

### 2.2 大V 投顾的问题

大V 投顾的核心问题并不是“没有内容”，而是“内容生产和经营链条太散”：

- 复盘内容生产成本高，且很难高频稳定输出。
- 粉丝消息、群发提醒、日常答疑、复盘发布之间没有统一后台。
- 历史方法论、行业框架、个股跟踪记录难以沉淀为可复用资产。
- 需要 AI，但不能是普通聊天框，而应当是贴合投研工作流的智能助手。
- 缺少经营 Dashboard，无法持续监控粉丝增长、转化、收入、消息分布与内容产出趋势。

### 2.3 平台管理方的问题

平台管理方关心的不是单次分析，而是整个平台的可控性：

- 租户之间必须隔离，方法论与知识资产不能混用。
- 某些能力必须按阶段控开，而不能一次性开放全部模块。
- Hermes、知识库、消息、工作台、会员、社区等能力要保持前后台一致口径。
- 内容输出需要证据链、风险边界和审计可追溯性。
- 平台需要把内容生产、粉丝经营、消息触达和商业化能力纳入统一管理框架。

## 3. 产品定位

### 3.1 一句话定位

洞见智研是一套面向证券研究与投顾经营场景的智能投研工作流平台，通过知识库、Skills 和 Hermes，把研究生产、内容分发、粉丝经营和平台治理连接起来。

### 3.2 当前阶段定位

当前阶段不是做一个覆盖所有金融人群的“大平台”，而是优先服务以下两类核心对象：

- 一类是需要结构化研究辅助的普通投资者。
- 一类是需要稳定生产研究内容、经营粉丝和沉淀方法论的大V 投顾租户。

平台 Admin 不是第三类用户市场，而是商业与治理体系成立的必要中枢。

### 3.3 核心产品边界

当前版本明确收敛在以下高频场景：

- 基本面分析
- 自选股与个股详情跟踪
- 复盘生产与复盘消费
- 知识库沉淀与微调
- Hermes 智能对话与工作区分析
- 粉丝消息与群发触达
- 大V 工作台经营 Dashboard
- 纯 Admin 的能力控开与治理

不做的事情同样重要：

- 不做高风险直接荐股表达。
- 不做泛社交内容平台。
- 不做无差别的通用 AI 问答工具。
- 不在当前阶段追求机构级复杂量化平台能力。

定位：面向普通投资者的轻量投研入口。

当前核心功能：

当前核心功能：

- 粉丝管理
- 粉丝消息
- 群发助手
- 经营 Dashboard
- 复盘生产台
- 知识库微调

- 自选股与详情

- 大V 经营主线：粉丝、消息、群发、发布、经营 Dashboard。
- 平台治理主线：Admin、控开、审计、一致性、租户隔离。
- 知识智能主线：知识库、Skills、Hermes 上下文与证据链输出。

## 5. 核心差异化能力

### 5.1 知识库是系统级能力，不是附件功能

这是项目最重要的独有能力之一。

知识库当前已经具备以下产品表达：

- 支持语音、文件、URL 等多入口沉淀内容。
- 历史入库内容可以再次选中、弹框微调与修改。
- 修改后内容会重新同步到知识专区与 Hermes 上下文。
- 知识条目可服务于复盘、个股分析、基本面判断与证据链归因。
- 知识库以租户为单位隔离，适合沉淀大V 的长期研究资产。

知识库的价值不在于“存资料”，而在于让资料成为可被 AI 调用、可继续微调、可嵌入工作流的生产资产。

### 5.2 Skills 让 AI 从“会回答”变成“会工作”

系统里的 Skills 不是简单标签，而是智能工作流的调度入口。

当前 Skills 已具备以下能力表达：

- Skills 可切换。
- Skills 可由大V 自定义新增。
- 每个 Skill 可绑定关联知识库。
- 每个 Skill 可绑定自己的提示词、研究框架与输出结构。
- Hermes 在回答时会自动继承当前 Skill 的方法视角。

这意味着 Hermes 不是一次性对话机器人，而是可长期复用、可持续打磨、可因租户而异的研究助手。

### 5.3 Hermes 双模式

Hermes 当前具备两种产品模式：

#### 工作区版

适用于较强结构化分析场景：

- 可先输入股票代码
- 可选择 Skill
- 可参考预设提示词
- 输出图表 + 内容 + 指标 + 证据链

#### 龙虾纯对话版

适用于更自然、更轻量的高级对话场景：

- 不强制单独输入股票代码
- 只保留 Skills 与对话
- 自动承接当前知识库上下文
- 对用户更像 ChatBI / 高级智能体

双模式的商业价值在于：

- 既能适应演示与专业用户，也能适应普通用户与轻量交互。
- 能让系统在“结构化工作流”与“自然对话体验”之间同时成立。

### 5.4 复盘不是文章页，而是工作流

当前复盘能力已经从“内容展示”升级为“内容生产系统”：

- 大V 可通过语音口述、手动撰写、文件上传来提供素材。
- 可先从大量自选股中挑选本次纳入复盘的股票。
- 可按行业板块复盘与个股投资复盘双层归纳。
- 可由大V 自定段落，也可交给智能体先成稿。
- 发布后可同步给粉丝，并进入消息链路。

这意味着复盘能力不仅带来内容消费价值，更直接形成大V 的生产效率和商业化入口。

### 5.5 核心指标面板支持单格智能生成

这是一个非常适合做差异化的高频功能。

当前产品逻辑是：

- 默认不展示编辑态，而是展示总结态。
- 若没有指标，则以加号形式展示。
- 用户点击单个格子后弹框输入提示词。
- 保存并关闭后，系统只针对该格进行生成。

这比“一整页重生成”更轻量、更可控，也更符合证券研究中局部跟踪、单指标复核的使用习惯。

### 5.6 从知识沉淀到超级智能体的护城河

本项目的长期护城河，不应被理解成“某个模型更强”，而应理解成一个持续积累、持续蒸馏、持续增强的金融认知系统。

#### a. 大V 的知识

每个大V 在长期研究、复盘、行业跟踪、个股观察和内容生产中，都会形成自己的判断框架、表达习惯与验证路径。  
这些内容通过知识库、复盘生产台、Skills 和 Hermes 工作流沉淀下来，构成租户级别的专属知识资产。

#### b. 粉丝群体的投资经验

平台沉淀的不只是大V 输出，还包括粉丝在互动过程中暴露出的投资经验：

- 提问方式
- 追问重点
- 对行情的理解偏差
- 实际犯错的共性
- 成功经验的共性
- 对某类行业或标的的长期观察点

这些内容会通过消息链路、答疑、复盘反馈和 Hermes 追问，不断沉淀为“粉丝群体经验层”。

#### c. 平台级跨租户知识归纳

在租户隔离前提下，平台管理方可以在更高抽象层做归纳总结：

- 哪类复盘框架更容易形成留存
- 哪类提示词更容易形成高质量回答
- 哪类问题最能代表散户真实需求
- 哪类知识结构更适合形成长期跟踪资产

这意味着平台并不只是保存单个大V 的知识，而是在制度允许和隔离规则清晰的前提下，逐步形成跨大V、跨粉丝群体、跨研究动作的元知识体系。

#### d. 可信数据源

平台护城河还包括底层可信数据源支撑。  
系统不是凭空生成观点，而是建立在可信研究资料、市场数据、公开信息、历史复盘资料和结构化知识条目之上，用数据源增强结论可信度、证据链完整性和审计可追溯性。

#### e. Hermes 智能体能力

Hermes 是知识调用、图表输出、证据链组织、对话追问和结构化表达的核心执行层。  
没有 Hermes，知识库只是资料；没有知识库和 Skills，Hermes 又会退化成普通聊天工具。二者结合才构成真正可复用的智能工作流能力。

#### f. 持续蒸馏大V 与粉丝群体认知

长期最重要的能力，不是回答一次问题，而是持续蒸馏：

- 大V 的成功经验
- 大V 的判断失误
- 粉丝群体的常见误区
- 粉丝群体的有效学习路径
- 长期复盘中反复出现的验证节点
- 行业、板块、个股在不同周期中的认知演化

这会让平台逐步形成一个“研究认知蒸馏系统”。

最终目标不是只有一个更会回答问题的金融 AI，而是形成一个不断吸收大V 方法论、粉丝经验、复盘历史、数据证据与平台治理经验的金融科技超级智能体。

## 5.7 当前阶段的知识飞轮

当前系统已经具备形成知识飞轮的基础结构：

1. 大V 生产复盘、答疑、研究框架和知识条目。  
2. 粉丝消费内容、提出问题、继续追问并反馈理解偏差。  
3. Hermes 在 Skills 和知识库约束下输出结构化分析。  
4. 平台沉淀每轮互动中的有效提示词、有效框架、常见误区和高价值结论。  
5. 大V 继续微调知识库与 Skills，推动下一轮更高质量输出。  

一旦这个飞轮跑起来，平台积累的就不是静态内容，而是越来越强的研究认知系统。

## 7. 商业模式设计

### 7.1 总体思路

本项目的商业模式不应只依赖单一付费路径，而应采用“三层收入结构”：

- 第一层：C 端个人付费
- 第二层：大V / 租户经营付费
- 第三层：机构与定制化付费

这样可以同时满足：

- 产品早期快速验证支付意愿
- 中期建立更稳定的 B2B2C 收入
- 后期通过高客单价提升收入上限

### 7.2 第一层：C 端个人付费

适合当前已具备的功能路径，主要包括：

#### 会员订阅

提供内容与功能层级差异，例如：

- 基础版：有限次数 Hermes 使用、查看普通复盘
- 专业版：更高频 Hermes 分析、更多知识引用、更多大V 内容
- 高级版：深度复盘、证据链模式、高价值消息与专题分析

#### 算力包 / Credits

系统已经具备积分与算力包的产品表达，适合进一步商品化：

- Hermes 基础算力包
- Hermes 专业算力包
- Hermes 量化算力包

该模式的优点：

- 用户更容易理解与接受
- 与 Token 消耗直接挂钩
- 便于做增长活动和积分兑换

#### 付费内容解锁

大V 回复、专题解读、闭门复盘等可以按篇或按专题解锁。

适合场景：

- 付费问答
- 付费复盘
- 专题行业框架
- 闭门交流纪要

## 9. 市场进入策略

### 9.1 第一阶段：用大V 场景建立真实使用频率

当前最优切入点不是直接面向海量散户做买量，而是先抓住一批“会持续产出研究内容”的大V 和研究型创作者。

原因：

- 大V 有更明确的高频复盘需求。
- 大V 更需要粉丝消息、内容生产、知识沉淀与经营后台。
- 大V 的使用行为天然带来普通投资者流量，是最初切入 C 端粉丝客群的核心入口。
- 大V 更容易形成付费、分发与转化闭环。

第一阶段目标：

- 打磨 10-30 个种子型大V 样本。
- 让他们稳定使用复盘生产台、知识库与 Hermes。
- 验证大V 对工作台的留存、续费与粉丝转化效果。

在这一阶段，平台的实际增长逻辑不是“先有大量 C 端，再去找大V”，而是：

1. 先服务有内容生产能力和粉丝基础的大V。  
2. 用复盘、答疑、Hermes、知识库和消息能力提升其内容生产与经营效率。  
3. 再把这些内容和互动承接到 H5，持续吸纳该大V 原有粉丝与新增粉丝。  

这条路径本质上是 B2B2C，而不是单纯 C 端买量。

### 9.1.1 大V 新媒体引流能力

除了承接已有粉丝，平台还应支持大V 用系统能力反向去各大新媒体渠道引流。

适合的渠道包括：

- 微信社群与私域
- 视频号
- 抖音
- 小红书
- 公众号
- 微博
- 直播与闭门交流活动

平台可为大V 提供以下引流抓手：

- 复盘内容二次分发
- 今日解读与群发提醒
- Hermes 结构化图表结果截图与摘要
- 专题知识卡片
- 付费答疑预览
- 复盘发布后的消息回流

长期看，平台不只是承接大V 现有粉丝，更会成为大V 在外部新媒体渠道做持续内容引流与粉丝沉淀的基础设施。

### 9.2 第二阶段：通过内容与消息链路转化普通投资者

普通投资者端的增长，不应单独看注册量，而应关注以下路径：

- 看见大V 复盘
- 进入 H5
- 完成基本面分析或自选股跟踪
- 使用 Hermes
- 加入消息链路
- 产生首次付费
- 形成高频留存

重点不是把入口做复杂，而是让高频动作极简：

- 输股票代码
- 看 K 线与 5/10/20 日线
- 看结构化分析
- 跟大V继续问
- 收到复盘更新

### 9.3 第三阶段：放大知识库与 Skills 的机构价值

当租户知识经营模式成立后，再进入更高客单价市场：

- 研究团队内部知识库搭建
- 机构版 Hermes
- 行业复盘协作工具
- 定制化 API 与私有工作流

## 12. 竞争与壁垒

### 12.1 相比通用大模型聊天产品

优势在于：

- 有明确的研究场景，而不是泛问答。
- 有知识库，不依赖单次上下文。
- 有 Skills，不只是 prompt。
- 有工作台，不只是聊天页面。
- 有粉丝经营与消息链路，不只是内容回答。
- 有 Admin 治理体系，不只是一个前台工具。

### 12.2 相比内容社区或投顾内容平台

优势在于：

- 强调内容生产工具链，而非单纯内容分发。
- 强调知识沉淀与智能复用，而非只发文章。
- 强调租户隔离与后台治理，而非统一内容池。
- 强调复盘、指标面板、个股详情与 Hermes 的联动。

### 12.3 相比传统金融 SaaS

优势在于：

- 更轻、更快、更贴近一线投研内容生产。
- 从真实高频动作切入，而不是从复杂系统切入。
- 更容易以低摩擦方式进入大V 和中小研究团队场景。

### 12.4 真实壁垒是什么

本项目真正的壁垒不是单个模型，也不是页面数量，而是以下六层能力叠加：

- 大V 方法论、长期复盘和行业框架形成的租户级知识资产
- 粉丝群体在互动、追问、犯错与成功经验中沉淀出的群体经验层
- 平台在隔离规则下提炼跨大V、跨粉丝群体、跨场景的元知识体系
- 可信数据源、研究资料与证据链支撑的底层可信性
- Hermes 作为执行层，把知识、Skills、图表与对话真正转成工作流
- 持续蒸馏大V 与粉丝群体认知，长期演化为金融科技超级智能体

换句话说，平台的长期价值不只是“谁能回答”，而是谁能长期沉淀、组织、验证、蒸馏并复用整个平台里不断增长的研究认知。

## 16. 风险与应对

### 16.1 风险一：AI 输出质量不稳定

应对：

- 强化知识库和 Skills 绑定，减少纯生成自由度
- 强化证据链、来源条目与风险提示
- 优先打磨少数高频模式，而不是盲目扩展模式数

### 16.2 风险二：大V 不愿意迁移工作流

应对：

- 先服务高频复盘型大V
- 用复盘生产台、消息中心和知识库微调作为核心切入点
- 用经营 Dashboard 证明对粉丝经营有实际价值

### 16.3 风险三：普通用户只尝鲜不留存

应对：

- 让自选股、个股详情与 Hermes 更低门槛
- 提高消息触达与复盘提醒频率
- 引导用户围绕固定研究动作形成习惯

### 16.4 风险四：商业模式过早分散

应对：

- 先集中验证 C 端订阅、大V 工作台与算力包
- 定制化与机构收入作为第二阶段目标
- 保持产品边界收敛

### 16.5 风险五：前后台口径失真

应对：

- 持续把一致性设计作为产品能力本身
- 保持首页、H5、工作台、Admin 同一套表达
- 使用统一能力矩阵与控开逻辑

- 用户分析入口
- 大V 复盘生产
- 消息触达
- 知识资产沉淀
- Skills 绑定
- Hermes 调用
- Admin 治理与控开

## 18. 结论

从当前功能现状看，洞见智研已经具备一个可继续深挖的清晰商业方向：

- 前台有足够轻的研究入口。
- 大V 工作台有足够明确的生产和经营价值。
- Admin 后台有足够清晰的治理与控开逻辑。
- 知识库 + Skills + Hermes 已经构成真正差异化的底层架构。

接下来最重要的，不是继续无边界加功能，而是围绕以下三件事把验证做深：

1. 让大V 工作台成为真正被高频使用的研究与经营后台。  
2. 让普通投资者围绕自选股、复盘和 Hermes 形成持续使用与付费。  
3. 让知识库与 Skills 成为租户不可替代的长期资产。  

如果这三件事成立，洞见智研就不仅是一个原型完整的产品，而是一套具备持续收入、可复制租户模型和长期壁垒的智能投研平台。
