## 1. 我们在做什么

洞见智研是一套面向证券研究与投顾经营场景的智能投研工作流平台。

它不是泛金融 SaaS，也不是普通聊天 AI，而是围绕以下高频动作构建：

- 基本面分析
- 自选股跟踪
- 日/周/月复盘
- 知识库沉淀与微调
- Skills 驱动的智能研究能力
- Hermes 智能体输出
- 大V 粉丝经营与消息触达
- 平台级租户治理与能力控开

一句话概括：

**用知识库 + Skills + Hermes，把研究内容生产、内容分发、粉丝经营和平台治理连成一个闭环。**

---

## 2. 为什么现在值得做

当前市场出现了一个明显空档：

- 用户已经接受 AI 辅助研究，但不再满足于普通聊天工具。
- 大V 和研究型创作者需要更强的复盘、答疑、知识沉淀和粉丝经营工具。
- 平台管理方需要前后台一致、租户隔离、能力可控开的系统，而不是散乱的单点工具。

通用 AI 的问题是“会回答但不会工作”，传统内容平台的问题是“会分发但不会沉淀知识”，传统 SaaS 的问题是“太重、太慢、离一线场景太远”。

洞见智研切入的是这三者之间最实际的一层：**高频研究动作工作流。**

---

## 3. 用户痛点

### 普通投资者

- 信息分散，难以整理
- 看不懂长篇研究内容
- 自选股很多，但没有统一跟踪逻辑
- 对 AI 的可信度、来源和边界存疑
- 没有形成长期研究习惯

### 大V 投顾

- 复盘和内容生产成本高
- 粉丝消息、群发提醒、复盘发布分散
- 方法论和历史资料难沉淀
- 需要 AI，但不能只是一个聊天框
- 缺少经营 Dashboard 和后台体系

### 平台管理方

- 需要租户隔离
- 需要能力分阶段控开
- 需要前后台一致口径
- 需要证据链、审计和治理能力

## 4. 我们的切入路径

项目最初不是从海量 C 端买量切入，而是从 **大V 作为切入点** 开始。

路径是：

1. 先服务有内容能力和粉丝基础的大V。  
2. 用复盘生产、知识库、Skills、Hermes 和消息经营提升其效率。  
3. 再通过这些大V 的内容和互动，承接其原有粉丝与新增粉丝。  
4. 将这些 C 端用户沉淀为长期可经营的研究型用户资产。  

这条路径本质上是 **B2B2C**：

- B 端是大V / 投顾租户
- C 端是其粉丝与普通投资者
- 平台通过工作台与 H5 同时建立商业闭环

## 6. 核心差异化能力

### 6.1 知识库不是附件，而是底层系统

知识库支持：

- 语音
- 文件
- URL
- 历史内容微调
- 同步到 Hermes 上下文

价值不在“存资料”，而在于把资料变成可被智能体持续调用的生产资产。

### 6.2 Skills 让 AI 从问答变成工作流

Skills 当前具备：

- 可切换
- 可自定义新增
- 可绑定知识库
- 可绑定提示词与输出结构

所以 Hermes 输出的不是随机聊天结果，而是有方法论约束的结构化分析。

### 6.3 Hermes 双模式

- 工作区版：股票代码 + Skill + 提示词 + 图表 + 证据链
- 龙虾纯对话版：只保留对话和 Skill，上下文自动带入知识库

这让系统兼顾专业分析和轻量对话。

### 6.4 复盘生产工作流

复盘支持：

- 语音口述
- 手动撰写
- 文件上传
- 先选股再归纳
- 行业板块复盘 + 个股投资复盘
- 大V 自定段落 / 智能体成稿

这意味着复盘是生产系统，不是静态文章页。

### 6.5 核心指标面板

支持单格提示词生成：

- 默认总结态
- 空态显示加号
- 点击单格后输入独立提示词
- 保存后只生成该格

这是非常贴近证券研究日常工作的轻量能力。

## 7. 我们真正的护城河

项目真正的壁垒不是单个模型，而是一个持续增强的金融认知系统。

### 7.1 第一层：大V 的知识

大V 在长期复盘、行业跟踪、个股观察和内容输出中，形成自己的研究方法论。  
这些内容通过知识库、复盘与 Skills 形成租户级专属资产。

### 7.2 第二层：粉丝群体的投资经验

平台沉淀的不只是大V 输出，还有粉丝在互动中暴露出的：

- 常见误区
- 追问重点
- 成功经验
- 失败经验
- 长期观察点

这会形成粉丝群体经验层。

### 7.3 第三层：平台级元知识归纳

在租户隔离前提下，平台可以抽象出：

- 哪类框架更容易形成留存
- 哪类提示词更容易形成高质量回答
- 哪类问题最代表真实需求
- 哪类知识结构更适合长期沉淀

这会形成平台级元知识体系。

### 7.4 第四层：可信数据源

系统建立在可信研究资料、市场数据、公开信息、历史复盘与结构化知识条目之上，增强：

- 结论可信度
- 证据链完整性
- 审计可追溯性

### 7.5 第五层：Hermes 智能体能力

Hermes 负责把知识、Skills、图表、证据链和对话真正转成可复用工作流。  
没有 Hermes，知识只是资料；没有知识库与 Skills，Hermes 只是聊天框。

### 7.6 第六层：认知蒸馏

平台长期蒸馏：

- 大V 的成功经验
- 大V 的判断失误
- 粉丝群体的认知偏差
- 长期复盘中的重复验证节点

长期目标是形成一个不断吸收大V 方法论、粉丝经验、复盘历史与数据证据的 **金融科技超级智能体**。

## 10. GTM 与增长路径

### 第一阶段：大V 样本验证

目标：

- 打磨 10-30 个种子型大V
- 验证工作台留存
- 验证知识库与 Hermes 使用频率
- 验证粉丝转化能力

### 第二阶段：通过内容和消息承接 C 端

普通投资者增长路径：

- 看见大V 复盘
- 进入 H5
- 完成分析或自选跟踪
- 使用 Hermes
- 加入消息链路
- 产生首次付费
- 形成高频留存

### 第三阶段：大V 反向做新媒体引流

平台支持大V 在外部渠道持续引流：

- 微信私域
- 视频号
- 抖音
- 小红书
- 公众号
- 微博
- 直播和闭门活动

可输出素材包括：

- 复盘内容二次分发
- Hermes 图表摘要
- 专题知识卡片
- 付费答疑预览
- 群发提醒与消息回流

### 第四阶段：机构化放大

- 研究团队知识库建设
- 机构版 Hermes
- 行业复盘协作工具
- 定制化 API 与私有工作流

## 11. 当前阶段验证重点

未来 6-12 个月重点验证三件事：

### 11.1 用户是否愿意为研究效率持续付费

不是只看首单，而是看复购、续费和高频留存。

### 11.2 大V 是否愿意把日常经营迁移到工作台

重点看：

- 复盘生产使用率
- 知识库维护频率
- Skills 创建率
- 消息中心与群发工具使用率

### 11.3 知识库 + Skills 是否构成可复制壁垒

重点看：

- 租户知识积累
- 自定义 Skill 数量
- 不同租户方法论差异化
- 回答质量稳定性

## 14. 主要风险与应对

### 风险一：AI 输出质量不稳定

应对：

- 强化知识库和 Skills 绑定
- 强化证据链与风险提示
- 先打磨少数高频模式

### 风险二：大V 不愿迁移工作流

应对：

- 先服务高频复盘型大V
- 用复盘、消息、知识库切入
- 用 Dashboard 证明经营价值

### 风险三：C 端只尝鲜不留存

应对：

- 降低自选股和 Hermes 门槛
- 强化复盘提醒与消息回流
- 围绕固定研究动作建立习惯

## 15. 为什么这会成为一门值得投的生意

如果我们只做一个前台 AI 工具，这不是一门足够大的生意。  
如果我们只做一个后台 SaaS，这也很难快速切入真实场景。

但现在我们已经把以下链路连接起来：

- 前台研究入口
- 大V 复盘生产
- 粉丝消息触达
- 知识资产沉淀
- Skills 结构化调用
- Hermes 智能执行
- Admin 治理与控开

这意味着洞见智研不是单点工具，而是一套具备长期知识壁垒、经营闭环和多层收入模型的智能投研工作流平台。

---

## 16. 结论

对投资人而言，这个项目最值得关注的不是功能数量，而是三件事：

1. 是否能用大V 切入，低成本承接 C 端粉丝客群。  
2. 是否能用知识库 + Skills + Hermes 建立真正不可替代的认知壁垒。  
3. 是否能在 C 端付费、大V 工作台收费和机构试点之间形成可复制的收入结构。  

如果这三件事成立，洞见智研将不是一个演示型 AI 产品，而是一套具备持续收入、长期知识飞轮和超级智能体潜力的金融科技平台。
