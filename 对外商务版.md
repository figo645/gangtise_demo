## 1. 我们是谁

洞见智研是一套面向证券研究与投顾经营场景的智能投研工作流平台。

我们聚焦的不是泛聊天，也不是泛内容分发，而是把真正高频的研究动作与经营动作组织起来，包括：

- 基本面分析
- 自选股跟踪
- 日/周/月复盘
- 知识库沉淀与微调
- Skills 驱动的智能分析
- Hermes 智能体对话与工作区分析
- 粉丝消息与群发触达
- 大V 工作台经营 Dashboard

一句话概括：

**帮助大V、研究团队和合作方，把内容生产、知识沉淀、粉丝经营和智能分析放进同一套系统里。**

## 2. 适合谁合作

### 2.1 大V / 投顾 / 研究型创作者

适合这些人：

- 有稳定复盘输出需求
- 有粉丝和私域经营需求
- 希望把内容生产流程标准化
- 希望把自己的研究方法沉淀成长期资产
- 希望借助 AI 提升效率，但又不想只是用一个普通聊天框

### 2.2 合作渠道 / 新媒体伙伴

适合这些对象：

- 需要稳定研究内容供给的内容渠道
- 希望合作大V 做专题内容和活动转化的机构
- 有私域用户、社区用户或投研受众的流量方

### 2.3 机构试点方

适合这些团队：

- 小型研究团队
- 投顾团队
- 买方研究辅助团队
- 希望建设私有知识库与智能工作流的团队

### 3.1 对大V 而言

当前大V 工作台 Web 已具备：

- 粉丝管理
- 粉丝消息
- 群发助手
- 经营 Dashboard

- 语音口述
- 手动撰写
- 文件上传
- 先选股后归纳
- 行业板块复盘
- 个股投资复盘

内容生产效率显著提升，尤其适合高频复盘型大V。

#### 更长期的知识资产

平台不只是帮大V“写一次内容”，而是帮大V 把长期研究过程沉淀成资产：

- 知识库
- Skills
- 复盘历史
- 粉丝反馈
- 常用提示词
- 表达模板

这些内容会越来越形成大V 自己的研究壁垒。

#### 更强的粉丝经营能力

系统支持：

- 粉丝消息管理
- 群发提醒
- 日常答疑
- 复盘发布推送
- 经营 Dashboard

大V 可以更清楚地知道：

- 哪类粉丝更活跃
- 哪类内容更能转化
- 哪类消息更值得跟进
- 哪类复盘更容易形成打开率和追问率

### 3.2 对合作渠道而言

渠道方可以得到：

- 稳定研究型内容供给
- 更标准化的复盘产出
- 更适合分发的图表化分析内容
- 更可复用的专题知识卡片
- 更适合做活动、答疑、转化的内容工具组合

### 3.3 对机构试点而言

机构可以得到：

## 4. 我们的核心能力

### 4.1 知识库

知识库支持：

- 语音
- 文件
- URL
- 历史内容微调
- 同步到 Hermes 上下文

这意味着合作方不只是把资料上传，而是把资料变成能持续参与分析的智能资产。

### 4.2 Skills

Skills 是把知识、提示词、结构化输出方式组织成可复用研究能力的模块。

当前已经支持：

- Skills 切换
- Skills 自定义新增
- Skills 绑定知识库
- Skills 绑定提示词和输出框架

### 4.3 Hermes

Hermes 是系统的智能执行层，当前有两种演示模式：

- 工作区版：适合股票、Skill、图表、证据链一起输出
- 龙虾纯对话版：适合更轻量、更自然的高级对话模式

Hermes 的价值不只是回答问题，而是以知识库和 Skills 为基础，给出图表加内容的结构化结果。

### 4.4 复盘生产台

复盘生产台支持：

- 语音口述
- 手动编辑
- 文件上传
- 选股归纳
- 行业复盘与个股复盘双层输出
- 大V 自定段落 / 智能体辅助成稿

### 4.5 核心指标面板

支持单格提示词驱动生成：

- 默认总结态
- 空态加号
- 单格独立编辑
- 保存后单格生成

这特别适合大V 或研究团队按主题持续跟踪关键指标。

## 5. 为什么我们和普通 AI 工具不同

### 普通聊天工具的问题

- 没有租户知识隔离
- 没有长期知识沉淀
- 没有 Skills
- 没有复盘工作流
- 没有粉丝经营后台
- 没有前后台一致治理

### 洞见智研的不同

- 有知识库
- 有 Skills
- 有 Hermes 双模式
- 有复盘生产台
- 有粉丝消息与群发
- 有经营 Dashboard
- 有 Admin 治理与控开

所以我们不是一个聊天框，而是一套完整的投研经营工作流。

## 11. 下一步怎么合作

如果是大V，可优先从以下路径开始：

1. 开通工作台试用  
2. 导入一批历史复盘或研究资料  
3. 建立第一批知识库和 Skills  
4. 跑通一次复盘生产与发布  
5. 接入粉丝消息与群发提醒  

如果是渠道方，可从以下路径开始：

1. 选择联合内容方向  
2. 定义复盘或专题内容形态  
3. 设计分发与转化路径  
4. 联合验证触达、打开率与付费效果  

如果是机构试点，可从以下路径开始：

1. 梳理现有研究资料  
2. 搭建私有知识库  
3. 配置专属 Skills  
4. 跑通 Hermes 与复盘工作流  
5. 评估后续定制化与长期合作模式

如果合作方希望得到的是：

- 更长期的知识沉淀

- 更容易复制的研究与经营体系

那么洞见智研适合成为这套体系的底层基础设施。
